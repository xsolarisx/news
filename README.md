# News Digest

A command-line news aggregator that fetches RSS feeds and scraped content from 18+ international sources, categorizes stories, deduplicates them, scores them by importance, detects language, translates non-English content, generates AI-written executive summaries per category, tags sentiment, and delivers output to the terminal, an HTML newsletter, a Markdown file, or email.

---

## Features

- **Async RSS fetching** via `aiohttp` — all feeds fetched concurrently (~5x faster than sequential)
- **SQLite caching** with configurable TTL — avoids redundant network requests across runs
- **TOML configuration** — all settings in `config.toml`; no source-code edits required
- **Two-stage deduplication** — global prefix match (first 7 title tokens) + per-category Jaccard similarity (threshold 0.5)
- **Story importance scoring** — `source_count` field tracks how many independent sources cover the same story
- **Language auto-detection** via `langdetect`
- **Translation** — non-English titles and summaries translated to English via OpenAI `gpt-4o-mini`
- **AI executive summaries** — one-paragraph category summaries generated by Anthropic `claude-haiku-4-5-20251001`
- **Sentiment tagging** — each story tagged positive, neutral, or negative using keyword matching
- **Multiple output modes** — rich terminal (color, bold), HTML newsletter, Markdown
- **Email delivery** — send the digest via SMTP using `smtplib`
- **Source health monitoring** — `--health` flag reports per-source fetch status and item counts
- **Hacker News integration** — top stories fetched from the HN JSON API

## Quick Start

```bash
git clone <repo>
cd news
pip install -r requirements.txt
cp config.toml.example config.toml   # edit as needed
python3 news_digest.py
```

## Installation

### Requirements

Python 3.9 or later.

```bash
pip install aiohttp feedparser rich langdetect anthropic requests
```

On Python versions below 3.11, also install:

```bash
pip install tomli
```

### Environment variables

| Variable | Purpose |
|---|---|
| `OPENAI_API_KEY` | Required for translation via `gpt-4o-mini` |
| `ANTHROPIC_API_KEY` | Required for AI executive summaries via Claude |

These can also be set in `config.toml` under `[ai]` or in the deployment environment file `/etc/news-digest/env`.

---

## Configuration

All runtime configuration lives in `config.toml` (or the path supplied with `--config`).

See [docs/configuration.md](docs/configuration.md) for the full reference.

A minimal example:

```toml
[digest]
max_per_cat = 8
similarity_threshold = 0.5
cache_ttl_seconds = 3600

[output]
width = 88

[ai]
openai_model = "gpt-4o-mini"
anthropic_model = "claude-haiku-4-5-20251001"

[email]
smtp_host = "smtp.example.com"
smtp_port = 587
smtp_user = "user@example.com"
smtp_password = "secret"
from_address = "digest@example.com"
to_addresses = ["you@example.com"]
```

## CLI Usage

```
python3 news_digest.py [OPTIONS]

Options:
  --category CAT        Show only stories matching this category name
  --output MODE         Output mode: terminal (default), html, markdown
  --output-file FILE    Write output to FILE instead of stdout
  --email               Send the digest via SMTP after rendering
  --no-cache            Skip the SQLite cache; fetch all feeds fresh
  --no-translate        Skip translation; show stories in original language
  --health              Print a source health report and exit
  --config FILE         Path to config file (default: config.toml)
```

### Examples

```bash
# Default terminal digest
python3 news_digest.py

# Tech stories only
python3 news_digest.py --category Tech

# Write HTML newsletter to file
python3 news_digest.py --output html --output-file /tmp/digest.html

# Write Markdown and send email
python3 news_digest.py --output markdown --email

# Force fresh fetch, skip translation
python3 news_digest.py --no-cache --no-translate

# Check source health
python3 news_digest.py --health
```

## Deployment

The tool runs in a Proxmox LXC container on Debian 12, triggered by a systemd timer at 06:00 and 18:00. Production files are installed at `/opt/news-digest/`.

See [docs/deployment.md](docs/deployment.md) for the full step-by-step guide.
